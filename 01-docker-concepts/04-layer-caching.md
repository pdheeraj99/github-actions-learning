## 7. Interview Answer ğŸ¯

```
Q: "How do you optimize Docker build time?"

A: "We order Dockerfile commands by change frequency.
    
    1. Least frequently changing commands at top:
       - Base image (FROM)
       - Package dependencies (package.json + npm ci)
    
    2. Most frequently changing at bottom:
       - Source code (COPY . .)
       - Build commands
    
    This maximizes layer cache hits. When only source code changes,
    npm ci layer is cached, saving 2-3 minutes per build.
    
    For 50 daily builds, this saves ~2.5 hours/day!"
```
